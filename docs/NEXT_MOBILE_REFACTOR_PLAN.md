# Booru Mobile Safari 后续修改计划

> 仓库：`aiza2a/booru-mobile-safari`
> 当前分支：`main`
> 当前定位：**仅面向 iPhone Safari / Stay 的手机端版本**

## 1. 当前已确认状态

### 1.1 项目边界
- 只保留 5 个站点：
  - Yande.re
  - Konachan.com
  - Konachan.net
  - Danbooru
  - Gelbooru
- 已删除或停用：
  - 下载、批量下载、URL 导出、FSA
  - Fancybox
  - 大量非目标站点兼容模块
- 当前桌面端仍会进入旧的桌面 UI 分支，但本项目目标不是继续适配桌面端，而是**明确限制为手机端专用**。

### 1.2 当前主要问题
1. **桌面端仍显示旧顶栏和旧设置面板**
   - 原因：`AppBar.vue` 和 `SettingsDrawer.vue` 仍保留 `!isMobile` 分支。
   - 应改为：桌面端不再渲染旧 UI，而是直接提示“本版本仅支持 iPhone Safari / Stay”。

2. **浅色模式未完成移动端视觉适配**
   - 当前浅色模式只是局部覆盖，和深色模式的一致性、层级感、透明度、对比度都不足。
   - 必须补齐到和深色模式相同的完成度，而不是“能看”。

3. **文档与源码定位还需进一步一致**
   - README 已改为手机端专用，但代码层仍存在桌面残留入口。
   - 需要让“项目定位”与“运行行为”完全一致。

---

## 2. 下一轮修改目标

下一轮只做以下两大块，不扩散：

### A. 桌面端彻底禁用旧 UI
目标：在电脑或非移动设备上，不再显示历史桌面版顶栏、侧栏和设置。

### B. 浅色模式补齐到与深色模式同等级
目标：手机端浅色模式的顶栏、胶囊、侧栏、设置面板、详情页都具备和深色模式一致的结构完成度与视觉层级。

---

## 3. 桌面端禁用方案

### 3.1 设计原则
- 不再继续维护桌面端适配。
- 不做“简易桌面版修补”。
- 不允许桌面端再进入旧的复杂分支逻辑。

### 3.2 推荐实现
在根布局或最上层容器增加统一判断：

- 若 `isMobile === true`：正常渲染手机端应用。
- 若 `isMobile === false`：渲染一个极简说明页。

说明页内容建议：
- 标题：`Booru Mobile Safari`
- 副标题：`此版本仅支持 iPhone Safari / Stay`
- 一段简短说明：
  - 本仓库为手机端精简版
  - 桌面端 UI 已不再维护
  - 请在 iPhone Safari / Stay 中使用
- 可附带安装脚本链接或仓库主页链接

### 3.3 修改范围建议
优先在：
- `src/App.vue`
- 或 `src/components/AppContainer.vue`

实现统一出口，避免继续到：
- `AppBar.vue` 的 `!isMobile` 分支
- `SettingsDrawer.vue` 的 `v-else` 桌面设置分支

### 3.4 验收标准
- 电脑打开时，不再显示旧桌面顶栏。
- 不再显示旧桌面设置面板。
- 页面只显示手机端专用提示。
- 手机端行为不受影响。

---

## 4. 浅色模式完善计划

## 4.1 目标
浅色模式应达到和深色模式同样的“完成态”，而不是局部改颜色。

### 必须保持一致的部分
- 顶栏整体布局
- 中间玻璃胶囊结构
- 左右独立悬浮按钮
- 搜索页、侧栏、设置面板的分组列表结构
- 图片详情页的背景、主图、底部操作栏结构
- Safe Area / 状态栏区域处理方式

---

## 4.2 浅色模式要统一的视觉变量
建议补一套更完整的浅色变量，而不是散落覆盖：

- 背景基色
- 次级背景
- 胶囊背景
- 独立按钮背景
- 分隔线颜色
- 主文本颜色
- 次文本颜色
- 高亮色
- 遮罩颜色
- 详情页暗化背景的浅色版

### 方向建议
浅色模式不要纯白、不要高对比硬边，保持 iOS 风格：
- 主背景：偏 `#f2f2f7`
- 卡片 / 胶囊：半透明白灰
- 分隔线：`rgba(60,60,67,.12~.18)`
- 文字：`#1c1c1e`
- 次要文字：`rgba(60,60,67,.78)`

---

## 4.3 顶栏浅色模式
需要补齐：
- 左菜单按钮
- 中间胶囊
- 右设置按钮
- 状态栏区域的过渡

要求：
- 与深色模式使用同一布局，不改结构。
- 只改材质、透明度和边框。
- 不允许浅色模式出现纯白硬块。
- 胶囊和按钮必须仍保留轻微玻璃感。

---

## 4.4 侧栏浅色模式
目标：与深色模式的分组结构和节奏完全一致。

要求：
- 保持当前已经整理过的对齐和间距。
- 图标、文字、当前站点高亮在浅色背景下仍清晰。
- 背景从深灰玻璃切换成浅灰玻璃，而不是纯白抽屉。
- 分隔线和标题灰度要自然。

---

## 4.5 设置面板浅色模式
当前设置面板结构已可用，但浅色模式需统一：
- 标题
- 分组标题
- 输入框
- 开关
- 下拉按钮
- 分隔线

要求：
- 不要出现部分区域是浅色、部分区域仍是深色残留。
- 输入框与按钮风格一致。
- 开关在浅色模式下对比度清晰。

---

## 4.6 图片详情页浅色模式
需要确认与补齐：
- 背景遮罩
- 模糊预览层
- 主图容器边框
- 底部操作栏

要求：
- 不因为浅色模式而出现发白、发灰或主图边缘不清。
- 主图仍是视觉中心。
- 底栏在浅色模式下依然有独立性，但不过度跳出。

---

## 4.7 状态栏与 Safe Area
浅色模式下尤其容易暴露背景不一致。

要求：
- 手机首屏顶部过渡要自然。
- 不出现深色残留线条。
- 不出现浅白硬条。
- 顶栏和状态栏下方内容的衔接要稳定。

---

## 5. 推荐执行顺序

### 阶段 1：桌面端禁用
1. 在最上层增加 `isMobile` 总开关。
2. 非移动端渲染提示页。
3. 手机端正常渲染。
4. 构建并验证。

### 阶段 2：浅色变量集中整理
1. 统一整理浅色变量。
2. 不改结构，只改颜色与材质。
3. 构建并验证。

### 阶段 3：按模块补齐浅色模式
顺序建议：
1. 顶栏
2. 侧栏
3. 设置面板
4. 搜索页
5. 图片详情

### 阶段 4：实机验收
在 Stay / iPhone 上至少检查：
- 首页深色
- 首页浅色
- 侧栏深色
- 侧栏浅色
- 设置面板深色
- 设置面板浅色
- 日期模式深色
- 日期模式浅色
- 图片详情深色
- 图片详情浅色

---

## 6. 严格边界
下一位执行者必须遵守：

- 不再恢复桌面端复杂交互。
- 不新增其他站点支持。
- 不恢复下载/FSA/Fancybox/批量下载。
- 不顺手改详情页结构、日期路由、分享逻辑、站点解析。
- 只做：
  - 桌面端禁用
  - 浅色模式补齐
